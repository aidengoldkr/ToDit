# ToDit Todo 생성 파이프라인 최적화 제안서

현재 To-Do 생성 파이프라인의 속도를 개선하기 위한 주요 기술적 전략들을 제안합니다.

## 1. LLM 응답 스트리밍 (Streaming) 도입 [가장 권장]
현재 시스템은 AI가 전체 결과(JSON)를 모두 생성할 때까지 사용자가 대기합니다.
- **제안:** OpenAI의 Streaming API를 사용하여 실시간으로 결과를 화면에 노출합니다.
- **효과:** 사용자가 첫 문장을 보는 시점(Time to First Token)이 대폭 단축되어 체감 속도가 수 초 이상 개선됩니다.

## 2. 데이터베이스 및 인증 로직 최적화
Lambda 내부에서 발생하는 여러 번의 DB 호출(티어 확인, 사용 조건 동의 확인, 사용량 제한 확인 등)이 순차적으로 실행되고 있습니다.
- **제안:** 
    - **쿼리 통합:** Supabase RPC(Stored Procedure)를 사용하여 한 번의 호출로 티어, 동의 여부, 사용량 체크를 모두 수행합니다.
    - **캐싱:** 사용량 제한이나 티어 정보와 같이 자주 바뀌지 않는 데이터는 Vercel KV나 Redis 같은 고성능 캐시에 저장하여 응답 속도를 높입니다.

## 3. 네트워크 및 스토리지 I/O 단축
파일 업로드 및 전달 과정에서의 불필요한 대기 시간을 줄입니다.
- **제안:**
    - **파일 직접 전달 (Base64):** 상대적으로 작은 이미지나 텍스트의 경우, Supabase Storage를 거치지 않고 직접 Lambda POST 바디에 실어 보냅니다. (S3 업로드/다운로드 오버헤드 제거)
    - **병렬 처리:** 프론트엔드에서 파일 업로드와 동시에 Lambda 호출용 인증 토큰(JWT)을 미리 받아옵니다.

## 4. OCR 및 문서 처리 효율화
- **제안:** 
    - **이미지 최적화:** 업로드 전 프론트엔드에서 이미지 크기를 리사이징/압축하여 전송 데이터 양을 줄입니다.
    - **OCR 병렬성 강화:** 다중 이미지 처리 시 Google Vision API 응답을 기다리는 동안 가능한 전처리 작업을 수행합니다.

## 5. Lambda 성능 및 구조 개선
- **제안:** 
    - **Cold Start 방지:** Lambda Warm-up 전략을 사용하거나, Provisioned Concurrency를 설정하여 대기 시간을 없앱니다.
    - **비동기 정리 작업:** 결과 저장 후 발생하는 임시 파일 삭제 등의 클린업 작업은 사용자에게 응답을 보낸 후에 비동기적으로 처리합니다.

---
위 제안들 중 **'응답 스트리밍'**과 **'DB 쿼리 통합'**이 가장 즉각적이고 큰 효과를 볼 수 있는 핵심 개선 포인트입니다.
