# ToDit 프로젝트 전체 코드 리뷰

이 문서는 **ToDit** 프로젝트의 구조와 작동 원리를 비전문가도 쉽게 이해할 수 있도록 정리한 전체 코드 리뷰 보고서입니다.

---

## 1. 프로젝트 개요 (Project Overview)

**ToDit**은 사용자가 업로드한 문서(이미지, PDF)나 직접 입력한 텍스트를 분석하여, 실행 가능한 **할 일 목록(To-Do List)**으로 변환해주는 AI 비서 서비스입니다.

- **핵심 가치**: "읽기만 하는 문서"를 "실행하는 단계"로 바꿉니다.
- **주요 기술**: 인공지능(OpenAI), 문자 인식(Google OCR), 클라우드 데이터베이스(Supabase).

---

## 2. 전체 흐름도 (Service Workflow)

사용자가 사진을 찍어 올릴 때부터 결과가 나오기까지의 과정입니다.

1. **입력 단계**: 사용자가 이미지, PDF 파일을 업로드하거나 텍스트를 입력합니다.
2. **글자 추출 (OCR)**: 이미지가 들어오면 Google AI가 사진 속 글자를 텍스트로 읽어냅니다.
3. **AI 분석 (OpenAI)**: 추출된 텍스트를 GPT-4 AI에 전달합니다. 이때 AI는 "할 일로 쪼개기(Decomposition)" 규칙을 따릅니다.
4. **결과 생성**: AI가 제목, 카테고리, 세부 할 일, 기한, 우선순위를 포함한 데이터를 만듭니다.
5. **저장 및 확인**: 결과는 데이터베이스에 저장되어 언제든 다시 볼 수 있으며, 내 대시보드에서 확인할 수 있습니다.

---

## 3. 주요 구성 요소 및 코드 로직 리뷰

### 3.1. 사용자 인터페이스 (Frontend)

사용자가 직접 눈으로 보고 상호작용하는 부분입니다.

- **랜딩 페이지 (`src/app/page.tsx`)**
  - 서비스의 첫인상을 결정합니다.
  - 구글 로그인을 유도하며, 서비스의 주요 기능(OCR, 지능형 To-Do 생성 등)을 시각적으로 설명합니다.
- **업로드 페이지 (`src/app/upload/page.tsx`)**
  - **드래그 앤 드롭**: 파일을 마우스로 끌어다 놓을 수 있는 편리한 인터페이스를 제공합니다.
  - **Pro 옵션**: 유료 사용자(Pro)는 '우선순위 분석', '상세도 설정', '사용자 지정 카테고리' 같은 고급 기능을 선택할 수 있습니다.
  - **진행 상태 표시**: AI가 분석하는 동안 사용자에게 로딩 애니메이션과 광고(무료 사용자용)를 보여줍니다.

### 3.2. 인공지능 엔진 (AI & OCR Logic)

프로젝트의 가장 중요한 "두뇌" 역할을 하는 부분입니다.

- **문자 인식 엔진 (`src/lib/google-ocr.ts`)**
  - Google Cloud Vision API를 사용합니다.
  - 사진 속의 손글씨나 복잡한 표에서도 정확하게 글자를 뽑아내는 핵심 로직입니다.
- **할 일 분해 로직 (`src/lib/openai.ts`)**
  - AI에게 단순히 "요약해줘"라고 하지 않고, 아주 구체적인 지침(System Prompt)을 줍니다.
  - **분해 규칙**: "3월 15일까지 수행평가"라는 문장이 있으면, "자료 조사하기", "PPT 만들기", "발표 연습하기" 등으로 단계를 나눕니다.
  - **어투 설정**: 모든 할 일은 한국어 동사구(`~하기`, `~제출하기`)로 끝나도록 강제하여 일관성을 유지합니다.
  - **날짜 계산**: 문서에 연도가 없으면 현재 연도를 기준으로 계산하고, 마감일에 맞춰 각 단계의 기한을 역산하여 배치합니다.

### 3.3. 데이터 및 API 관리 (Backend)

데이터를 처리하고 저장하는 "보이지 않는 일꾼"들입니다.

- **통합 파서 API (`src/app/api/parse/route.ts`)**
  - 모든 요청이 모이는 통로입니다.
  - 사용자 권한을 체크(로그인 여부, 이용약관 동의 등)합니다.
  - 유료/무료 등급에 따라 서로 다른 인공지능 모델(gpt-4o vs gpt-4o-mini)을 배정합니다.
- **사용량 및 구독 관리 (`src/lib/usage.ts`, `src/lib/subscription.ts`)**
  - 사용자가 한 달에 몇 번 서비스를 이용했는지 기록합니다.
  - 무료 사용자는 월 한도(예: 30회)가 있으며, 초과 시 업그레이드 안내를 보냅니다.
- **데이터베이스 (`src/lib/supabase/`)**
  - 사용자가 생성한 To-Do 목록을 안전하게 저장하고, 나중에 대시보드에서 다시 불러올 수 있게 합니다.

## 4. 사소하지만 중요한 로직들 (Hidden Details)

비전문가는 놓칠 수 있지만, 완성도를 높이는 세심한 처리들입니다.

1. **세션 저장 (`action-plan-session.ts`)**: 브라우저를 새로고침해도 분석 결과가 사라지지 않도록 임시로 데이터를 기억합니다.
2. **PDF 텍스트 추출**: 이미지뿐만 아니라 PDF 파일의 글자도 읽어와 분석에 포함합니다.
3. **연도 추론 규칙**: 날짜에 연도가 없을 때, 현재 날짜보다 6개월 이상 미래면 작년 문서로 판단하거나 하는 세밀한 예외 처리가 들어있습니다.
4. **보안 처리**: 사용자가 올린 파일은 분석이 끝나면 즉시 임시 저장소에서 삭제하여 개인정보를 보호합니다.

---

## 5. 결론

ToDit은 **"단순한 글자 읽기"**를 넘어 **"AI의 논리적 분석"**을 결합한 수준 높은 생산성 도구입니다. 복잡한 코드는 뒤편에 숨기고, 사용자는 사진 한 장으로 체계적인 일정을 얻을 수 있도록 설계되었습니다.
