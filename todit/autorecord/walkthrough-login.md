# 자체 로그인(Credentials) 구현 완료 보고서

지정된 계획에 따라 Google OAuth와 호환되는 자체 회원가입/로그인 기능을 성공적으로 구현하였습니다.

## 구현된 주요 사항

1.  **데이터베이스 스키마 확장**:
    -   `public.users` 테이블을 생성하는 마이그레이션 스크립트를 작성하여 이메일, 비밀번호 해시, 제공자(provider) 정보를 저장하도록 구성했습니다. (`supabase/migrations/` 및 `schema.sql`)
    -   `id` 컬럼을 `TEXT` 타입으로 설정하여 자체 가입자(UUID)와 Google 가입자(Google ID)를 모두 동일한 테이블에서 통합 관리할 수 있게 하였습니다.
2.  **NextAuth 인증 옵션 강화**:
    -   `bcryptjs`를 도입하여 비밀번호를 안전하게 해싱하도록 구현했습니다.
    -   `CredentialsProvider`를 추가하여 이메일 및 비밀번호 검증 로직을 작성했습니다.
    -   기존 Google 로그인 사용자 정보도 `users` 테이블에 자동으로 Upsert되도록 `signIn` 콜백 로직을 고도화했습니다. (계정 자동 연동)
3.  **커스텀 UI 컴포넌트 추가**:
    -   기존 디자인 시스템(`var(--surface)`, `var(--accent)` 등)을 정확히 준수하여 통일감 있는 모달형 카드 UI의 `/auth/signin` 및 `/auth/signup` 페이지를 개발했습니다.
    -   자체 로딩 상태, 오류 출력 처리(예: 잘못된 비밀번호, 중복 계정 등)를 내장하였습니다.
4.  **진입점 수정**:
    -   `Navbar.tsx`와 홈 페이지의 CTA 버튼들이 기존 구글 로그인 팝업 대신 새로 만든 `/auth/signin` 페이지로 이동하도록 연결했습니다.

## (필수) 수동 검증 단계 안내

완료된 기능의 정상 작동 확인을 위해 아래의 수동 테스트를 진행해 주시기 바랍니다.

### 1. 데이터베이스 마이그레이션 적용
작성된 새 마이그레이션 파일을 프로젝트의 Supabase 데이터베이스에 반영해야 합니다.
- **로컬/배포 환경**: 터미널에서 `npx supabase db push` 명령어를 실행하거나, Supabase 대시보드에서 `supabase/migrations/20260322_add_users_table.sql` 스크립트를 수동으로 실행해 주세요.

### 2. 회원가입 및 로그인 테스트
서버(`npm run dev`)를 실행한 뒤, 브라우저에서 다음 시나리오를 검증해 주세요.
- [ ] `/auth/signup` 에서 새로운 이메일로 가입 성공 및 대시보드 리다이렉트 확인 (DB에 해싱된 비밀번호 저장 확인).
- [ ] 로그아웃 후 `/auth/signin` 에서 방금 만든 계정으로 로그인 가능 여부 확인.
- [ ] 틀린 비밀번호 입력 시, 에러 메시지 빨간 박스가 UI에 표시되는지 확인.

### 3. 계정 연동 및 예외 테스트
- [ ] 이미 자체 회원가입된 이메일로 "Google로 계속하기" 버튼 클릭 시, 자동으로 기존 계정에 연동 및 로그인되는지 확인.
- [ ] 이미 Google로 가입된 이메일 주소를 입력하여 새 자체 회원가입 시도 시 에러가 뜨며 차단되는지 확인.

테스트 후 확인되는 이슈 사항은 즉시 피드백 주시면 수정하도록 하겠습니다.
