# Project Overview
ToDit is a Korean-language Next.js 14 (App Router) SaaS. (Migrated from Actonix).
**Core capability: Todo Decomposition** — high-level goals are decomposed into granular, executable sub-todos with a tree structure and reverse-calculated deadlines.
*   **Language Rule:** All internal reasoning can be in English, but **all UI text, user-facing responses, and final outputs MUST be in Korean.**

# Business Logic Invariants (STRICT RULES)
**DO NOT change without explicit product owner approval.**
1. Pipeline order is immutable: consent check → plan restriction → usage reservation → OCR → LLM → save → commit usage.
2. Usage reservation (free tier) must be released on ANY failure after reservation.
3. OCR failure prevents LLM call.
4. reserveFreeUsage() / releaseFreeUsage() are the single source of free-tier quota logic. No duplicates.
5. Free tier model = gpt-4o-mini. Pro tier model = gpt-4o. 
6. response_format: json_object is required. 
7. Files in parse-temp bucket MUST be deleted after every parse.
8. Auth chain: session → consent → business logic. No endpoints bypass this.
9. Priorities are generated by the LLM for all tiers but stripped from the result for free-tier users.
10. Storage path ownership must be validated before any download.
11. Every time you complete a task, you MUST trigger the auto-record skill to save the work log.
